7 most useful commands of git for beginners

December, 2020

let's quickly see, what is git?

Git is a version control system which is freely available. It helps in tracking your projects and keeping them organized.

prerequisites

You must have downloaded the git on your system, then just open the terminal and go to your git directory.

we are good to go now!

  1. git pull

    It will pull all the updates from the server into your local machine. By doing it, we are downloading all the changes( most likely done by your other team members) and making your code updated with the latest one present on the server. It's just like we sync our google accounts on different devices so that we are provided with updated information.

  2. git status

    It basically tells the state of your repository and would provide the following information:

    • It lets you know about your current branch in which you are working.

    • It enlists the modified files. It informs you if there are any changes in the existing files that are not yet committed to git.

    • It tells you about untracked files i.e. which are on your local machine and yet not committed to git.

  3. git diff

    It will enlist all changes in all files.The code which has been removed will be displayed in red.The new code which has been added to the file will be displayed in green.

    git diff [filename] :- It will mention changes only in that particular filename.

  4. git checkout

    It is most commonly used to switch to a different branch. Another most common use for checkout is when you want to restore the previous version of the file.

    how to restore the version of file?

    For e.g. you have done some changes in the file and save it, now if you check the git status it will tell you list of modified files and will give you 2 suggestions:

    • If you want to add this file in your next commit then use git add “filename”.
    • If you want to discard the changes, then use git checkout “filename”. By doing checkout this file, it’s previous version will be restored on your local machine.

  5. git add [filename]

    It tells Git that you want to add this particular file into your next commit. However, git add doesn't update the git repository until you run git commit.
    It’s like you are selecting the changes for which you will do a commit.(just like when you finalizes the products in the supermarket by putting it in your cart , for which you will pay the bill)

    For e.g. if there are 3 untracked files, fileA, fileB, fileC, but you only want to commit for fileA and fileB, so you first do git add fileA, then git add fileB, then do commit.

    git add . :- it will add all modified files at once.

  6. git commit -m 'message'

    It will commit all the changes added by git add command along with message.

  7. git push

    It will push all the committed code to the server.
